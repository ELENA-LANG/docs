# 0xD4 -- udiv dp

- **Category:** TripleOp
- **Enum:** `ByteCode::UDivDPN`
- **Operand(s):** `udiv dp:disp, n` (n = 1, 2, 4, 8)
- **Reads:** `sp[0]` (n bytes), `dp[disp]` (n bytes)
- **Writes:** `dp[disp]` (32-bit low word per the templates)

## Semantics
`temp := n << sp[0]; dp[disp] := dp[disp] / temp` interpreted as unsigned. Backend templates implement the 32-bit case.

## JIT (amd64)
**Selection rule:** dispatched through `loadDPNOp` -> `retrieveICode(scope, arg2)` on the n-width. Although the encoding allows n  in  {1, 2, 4, 8}, all four backends only emit the 32-bit template -- narrower / 64-bit widths fall through to the default slot.

**Template:** unsigned `div ecx` with `rdx = 0`; result low half lands in `eax`.

```asm
mov  rcx, [r10]
xor  edx, edx
mov  rax, [rbp + __arg32_1]
div  ecx
mov  dword ptr [rbp + __arg32_1], eax
```

## JIT (x32)
**Template:** same idiom -- `xor edx, edx`, `div [esi]`.

```asm
xor  edx, edx
mov  eax, [ebp + __arg32_1]
div  dword ptr [esi]
mov  [ebp + __arg32_1], eax
```

## JIT (aarch64)
**Template:** `udiv` -- unsigned 32-bit divide.

```asm
add     x19, x29, __arg12_1
ldr     w17, [x0]
ldr     w18, [x19]
udiv    x18, x18, x17    ; sp[0] is divisor
str     w18, [x19]
```

## JIT (ppc64le)
**Template:** `divwu` -- unsigned word divide.

```asm
addi    r19, r31, __arg16_1
lwz     r17, 0(r3)
lwz     r18, 0(r19)
divwu   r18, r18, r17
stw     r18, 0(r19)
```

## Notes
- Distinct from `idiv dp:disp, n` (0xE4) -- signed divide.
- Division by zero traps via the host CPU exception (no software check).
- Although the spec advertises widths 1/2/4/8, all four backends only emit the 32-bit form -- wider/narrower variants are not generated by the current JIT (`retrieveICode` falls through to the default slot 0 for unsupported widths).
- amd64/x32 use `div` (unsigned); aarch64 uses `udiv`; ppc64le uses `divwu`.
- Result low half is stored at `[dp + disp]` (32-bit); the high half / remainder is discarded.
