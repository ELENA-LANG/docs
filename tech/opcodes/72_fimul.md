# 0x72 -- fimul

- **Category:** SingleOp
- **Enum:** `ByteCode::FIMul`
- **Operand(s):** (none)
- **Reads:** `acc` (pointer to double), `sp[0]` (pointer to int)
- **Writes:** `[acc]` (double)

## Semantics
In-place double-from-int multiply: `double:[acc] *= int:[sp[0]]`. The 32-bit integer at the address held in `sp[0]` is converted to double and multiplied with the double at the address held in `acc`.

## JIT (amd64)
**Template:** `asm/amd64/core60.asm` `inline %72h`

```asm
  mov  rax, r10
  fld   qword ptr [rbx]
  fild  [rax]
  fmulp
  fstp  qword ptr [rbx]
```

## Notes
- Same shape as `fiadd` (0x70), just with `fmulp` / `fmul`.
- Multiplication is symmetric, so the operand order divergence that affects `fisub` (0x71) and `fidiv` (0x73) does not apply here.
- amd64 prefixes the FP work with `mov rax, r10` to materialize the integer pointer in a base-friendly register before `fild`.
- Overflow produces +/-Inf per IEEE-754; no trap is generated by default.
- `index` and stack are preserved; only `[acc]` is updated.
